Deployment Configuration Guide for CyberSecured AI Platform

Prerequisites Setup

Step 1: VPC Networks Configuration

Network 1: Management Network

  • Network Name: cybersecured-ai-mgmt-network
  • Description: Management network for VM-Series firewall administration
  • Subnet Mode: Custom
Subnets:
  • Primary Subnet: cybersecured-ai-mgmt-subnet-primary
    • Region: us-central1
    • IP Range: 192.168.100.0/25
  • Secondary Subnet: cybersecured-ai-mgmt-subnet-secondary
    • Region: us-east1
    • IP Range: 192.168.100.128/25

Network 2: Untrust Network (Internet-Facing)

  • Network Name: cybersecured-ai-untrust-network
  • Description: Untrust network for internet-facing traffic
  • Subnet Mode: Custom
Subnets:
  • Primary Subnet: cybersecured-ai-untrust-subnet-primary
    • Region: us-central1
    • IP Range: 10.0.1.0/25
    • Flow Logs: On
  • Secondary Subnet: cybersecured-ai-untrust-subnet-secondary
    • Region: us-east1
    • IP Range: 10.0.1.128/25
    • Flow Logs: On

Network 3: Trust Network (Internal)

  • Network Name: cybersecured-ai-trust-network
  • Description: Trust network for internal application traffic
  • Subnet Mode: Custom
Subnets:
  • Primary Subnet: cybersecured-ai-trust-subnet-primary
    • Region: us-central1
    • IP Range: 10.0.2.0/25
    • Private Google Access: On
    • Flow Logs: On
  • Secondary Subnet: cybersecured-ai-trust-subnet-secondary
    • Region: us-east1
    • IP Range: 10.0.2.128/25
    • Private Google Access: On
    • Flow Logs: On

Step 2: Firewall Rules

Rule 1: Management Access

  • Name: allow-mgmt-cybersecured-ai
  • Direction: Ingress
  • Action: Allow
  • Network: cybersecured-ai-mgmt-network
  • Source IP ranges: 0.0.0.0/0
  • Protocols: TCP ports 22, 443, 3978

Rule 2: Web Application Traffic

  • Name: allow-web-cybersecured-ai
  • Direction: Ingress
  • Action: Allow
  • Network: cybersecured-ai-untrust-network
  • Source IP ranges: 0.0.0.0/0
  • Protocols: TCP ports 80, 443, 5000

Rule 3: Internal Communication

  • Name: allow-internal-cybersecured-ai
  • Direction: Ingress
  • Action: Allow
  • Network: cybersecured-ai-trust-network
  • Source IP ranges: 10.0.2.0/24
  • Protocols: TCP 1-65535, UDP 1-65535

Step 3: SSH Key Generation

Command:

ssh-keygen -t rsa -b 2048 -f ~/.ssh/cybersecured-ai-one -C "one@cybersecuredai.com"

To display public key:

cat ~/.ssh/cybersecured-ai-one.pub

Untrust IP

Post-Deployment Steps

Step 1: Verify Deployment

  • Check VM instance in Compute Engine
  • Note management IP address
  • Confirm all network interfaces are attached

Step 2: Access VM-Series Management

  1. Navigate to: https://[MANAGEMENT-IP]
  2. Default credentials: admin / admin
  3. Change password on first login
  4. Complete setup wizard

Step 3: License Configuration

  1. Go to Device → Licenses
  2. Enter BYOL license key
  3. Commit changes

Step 4: Deploy Application Server

gcloud compute instances create cybersecured-ai-app \
    --zone=us-central1-a \
    --machine-type=e2-standard-2 \
    --subnet=cybersecured-ai-trust-subnet-primary \
    --no-address \
    --image-family=ubuntu-2004-lts \
    --image-project=ubuntu-os-cloud

Security Architecture Overview

Traffic Flow:

Internet → VM-Series Firewall → Load Balancer → CyberSecured AI SaaS (Port 5000)

Three-Tier Network Security:

  • Management Network: Firewall administration
  • Untrust Network: Internet-facing traffic filtering
  • Trust Network: Protected application environment

Integration Benefits:

  • Enterprise-grade network security
  • Real-time threat intelligence integration
  • Automated policy management
  • Compliance reporting capabilities
